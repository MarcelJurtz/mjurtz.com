Android has various options for storing data internally, such as the integrated SQLite database. For testing purposes I created an application with Xamarin which stores data in a local NoSQL database. I use LiteDB for this purpose. However, this application became problematic when I tried to fix problems with my data structure. The database is stored in the Android file system, but it is not possible to open this path on a non-rooted device. This article is intended to document my solution, which can also be used for general access to app data.

First of all, ADB is needed, which is included in the Android SDK. A backup can then be created via ADB, provided the device is connected via USB. To create a backup this way, the name of the apps package is needed. To then create the backup, enter the following command: adb backup -noapk com.packackename. This results in an .ab-file (android backup) being created. I then found the Android Backup Extractor on Sourceforge, which is able to convert .ab-files to .tar-files, which again can be easily extracted. To convert the backup file, simply use this command: java.exe -jar abe.jar unpack backup.ab backup.tar "", which converts the backup.ab file to backup.tar.

After the extraction there are several files available, including the database file of litedb.
