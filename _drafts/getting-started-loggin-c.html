Today, I will cover logging in C# using NLog. NLog is a free logging platform for .NET, Xamarin and more and allows you to quickly add a logging functionality to your project.

To jump right into the topic, download the nuget packages NLog and NLog.config. The latter simply adds a config file to your project, in which you can filter your logs and configure where which logs will be stored. Let's start with setting up the configuration.

The default config file is pretty straight forward and describes itself quite well. In this article, I'd like to set up two log files: In the first one, all kinds of information will be saved. The second one will be used for exceptions and fatal errors. To be more specific: NLog differentiates between several log levels:

  • Fatal
  • Error
  • Warn
  • Info
  • Debug
  • Trace

The higher the position of an element in the list, the greater the prioritization level of corresponding messages. Nevertheless, you are completely free in the implementation of the actual logging. But let's get back to the config file.

Setup & Get started

At first, we will define several targets. A target is a location where log message can be stored by NLog. You can use a lot of different kinds of targets (you can also build custom ones), but we will stick to the simple 'File'. To try this out, we can add a simple target as shown below:

<targets>
  <target name="errorlog" xsi:type="File" fileName="errorlog.txt"/>
</targets>

The second important part of this setup is the filtering. To specify, whichÂ  kinds of logs should be written to which target, you will need to set up rules. The following rule states the logs, that are at least of the type 'Error' will be written to errorlog.txt:

<rules>
  <logger name="*" minlevel="Error" writeTo="errorlog" />
</rules>

That's all it takes for initial setup. By now, you should be able to log errormessages from any part of your program by applying the following syntax:

Logger logger = LogManager.GetCurrentClassLogger();
logger.Error("Hello World!");
